leerDocumentoTodasCategorias(self, G, numero, row_count, usu)

source code 

Lee la lista de seguidos del usuario que se pasa por parámetro, tras esto, en cada iteracción del bucle va añadiendo una relación nueva que se añade al grafo. Tras acabar la iteracción se sale de la función para hacer que el grafo vaya creciendo progresivamente. Cuando termina la lista de un usuario pone terminado a True y pasa al segundo nivel.

Parameters:
  • G (nx.Graph()) - Grafo al que añadiremos los nodos.
  • numero (int) - Para saber por donde vamos.
  • row_count (int) - Número de columnas en el archivo del usuario en cuestión.
  • usu (str) - Usuario del cual leemos sus seguidos.

sacarSeguidoresSegundoNivel(self, usuario)

source code 

Leemmos los seguidos del usuario pasado por parámetro. A partir de estos, se irán sacando las listas de los usuarios a segundo nivel. Una vez terminado, rompemos la interacción.

Parameters:
  • usuario (str) - Usuario a partir del cual leemos sus seguidos, y de estos sacamos sus seguidos.

leerDocumentoAlgunasCategorias(self, G, row_count, usu, checkAnimales, checkRopa, checkTerrorismo, checkComida, checkSc)

source code 

Lee la lista de seguidos del usuario que se pasa por parámetro, tras esto, se chequean que botones están activos para contar como una iteracción un usuario que cumpla los requisitos, si dicho usuario no los cumple pasa al siguiente, pero a diferencia de la función anterior, esta no se contaría como interacción. Cuando termina la lista de un usuario pone terminado a True y pasa al segundo nivel.

Parameters:
  • G (nx.Graph()) - Grafo al que añadiremos los nodos.
  • row_count (int) - Número de columnas en el archivo del usuario en cuestión.
  • usu (str) - Usuario del cual leemos sus seguidos.
  • checkAnimales (boolean) - Para saber si esta activado o no el checkBox de animales.
  • checkRopa (boolean) - Para saber si esta activado o no el checkBox de ropa.
  • checkTerrorismo (boolean) - Para saber si esta activado o no el checkBox de terrorismo.
  • checkComida (boolean) - Para saber si esta activado o no el checkBox de comida.
  • checkSc (boolean) - Para saber si esta activado o no el checkBox de sin calificar.

sacarSeguidoresSegundoNivelAlgunasCategorias(self, usuario, G)

source code 

Leemmos la lista de nodos que hay en G. Por cada iteracción iremos borrando de la lista el nodo consultado y sacando sus seguidos para posterior estudio.

Parameters:
  • usuario (str) - Usuario a partir del cual leemos sus seguidos, y de estos sacamos sus seguidos.
  • G (nx.Graph()) - Grafo al que consultamos sus nodos.

colorearNodos(self, G, color_map)

source code 

Se colorean los nodos en función del valor de los contoadores de cada usuario. Los colores serían:

  • Gris: Sin calificar.
  • Rojo: Terrorismo.
  • Verde: Animales.
  • Azul: Comida.
  • Amarillo: Ropa.
Parameters:
  • G (nx.Graph()) - Grafo al que consultamos sus nodos.
  • color_map (list) - Lista para colorear los nodos.

dibujar(self, checkAnimales, checkRopa, checkTerrorismo, checkComida, checkSc, manejoGrafico)

source code 

Esta función se divide en tres partes. La primera, si todos los checkBox están activados, en tal caso se llaman a las funciones leerDocumentoTodasCategorias y sacarSeguidoresSegundoNivel para el correcto funcionamiento del grafo. La segunda, no todos los checkBox están activados, en tal caso se llaman a las funciones leerDocumentoAlgunasCategorias y sacarSeguidoresSegundoNivelAlgunasCategorias para el correcto funcionamiento del grafo. Y por último, si no hay ningún checkBox activado, en tal caso se procede al error.

Parameters:
  • checkAnimales (boolean) - Para saber si esta activado o no el checkBox de animales.
  • checkRopa (boolean) - Para saber si esta activado o no el checkBox de ropa.
  • checkTerrorismo (boolean) - Para saber si esta activado o no el checkBox de terrorismo.
  • checkComida (boolean) - Para saber si esta activado o no el checkBox de comida.
  • checkSc (boolean) - Para saber si esta activado o no el checkBox de sin calificar.
  • manejoGrafico (object) - Ventana de manejoGrafico para manejar el comportamiento de sus componentes.
